## Operating contract per story (every commit)

- Conventional commit subject + omc-style trailers:
  ```
  Constraint: <which invariant or guardrail this commit respects>
  Rejected: <alternative considered and why not chosen>
  Confidence: HIGH | MED-HIGH | MED | MED-LOW | LOW
  Scope-risk: LOW | MED | HIGH
  ```
- One story → one commit (per principle 9). If a story spawns a multi-direction probe, each direction's investigation doc lands as its own commit in `docs/probes/`.
- After every commit: architect + critic independent review (per principle 4).
- Tag advance only after live smoke evidence is checked in (per principle 10).

## What this plan deliberately does NOT do

1. Does not chase macOS/Linux parity (locked deferral to v2.1+ per user 2026-05-25).
2. Does not push tags to remote before v2.0 (local-tag policy).
3. Does not bundle multiple stories per commit.
4. Does not pre-defer anything to v2.1 via ADR — every ADR in this plan is contingency, not pre-committed.
5. Does not collapse session N+1 work into N+2 to "save a session" — the deterministic foundation (det harnesses + agent matrix + doctor MCP check) materially de-risks N+2's live work.
