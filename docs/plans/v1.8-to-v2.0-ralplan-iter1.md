## ADR drafts (1 mandatory US-05 + 3 conditional templates)

### ADR-C1 (MANDATORY for v1.8): US-05 cost-governor outer-loop equivalent

- **Status**: Proposed (executes in N+2)
- **Decision**: Implement **Option C** — keep PostToolUse hook for accumulation; add a per-spawn post-spawn callback in `mode.ts` that writes `.omcp/state/cost-summary-<sessionId>.json` based on whichever of (a) Copilot stdout cost reporting in `-p` mode or (b) byte-count estimation is available.
- **Drivers**:
  - HUD column 6 dependency (v1.9 Tier 1).
  - v1.7 already deferred once (US-05 ADR-deferral); "no kick the can" hard rule.
  - Roadmap §v1.8 anti-pattern: silent re-deferral is forbidden.
- **Alternatives considered**:
  - **Option A** (per-spawn callback only): loses per-tool-call granularity forever, even after upstream Stop-hook fix.
  - **Option B** (new `cost-advice-emit` event only): adds a new event to Invariant 4's valid set; one more thing to keep in sync; doesn't preserve forward-compat with upstream fix.
  - **Option C** (both): best of both; pays small complexity cost.
- **Why C chosen**: forward-compatible with upstream fix; ships value now; preserves both code paths.
- **Consequences**: One new state file shape (`cost-summary-<sessionId>.json`); one new env var (`OMCP_SESSION_ID` threaded from `mode.ts`); double-counting risk addressed via test fixture in deterministic vitest.
- **Follow-ups**: HUD column 6 reads from the per-spawn summary in v1.9 (US-1.9-T1-HUD-column-6).
- **Invariants preserved**: I1 (safe-slug sessionId), I2 (atomicWriteFileSync at `src/hooks/cost-governor/index.ts:67` preserved).
