## Per-session commit budget + stop-and-replan rule (Section A)

Each session below carries an explicit **commit-budget ceiling**. If a session ends with **more than 8 stories incomplete** vs that session nominal target, the NEXT session first story is `replan-iter2-remaining` (which lands as a planning commit consuming a vitest +0 slot but writes a one-page "what carries forward, why" doc under `docs/plans/`). This is the systematic answer to architect#1 + critic H1.

**Effective story-vs-commit accounting**:
- 22 MCP det/e2e stories -> potentially 4 matrix commits (1 canary det, 1 remainder det, 1 canary e2e, 1 remainder e2e) OR more if per-server fixes land.
- 19 agent QA stories -> potentially 2 matrix commits (matrix-pass + per-agent-scrub commits as fixes land).
- Net result: N+1 nominal-story count = 42 but worst-case commit count = ~14-22.
