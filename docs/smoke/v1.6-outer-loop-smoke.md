## What this proves

1. **The outer loop drives multiple Copilot spawns** — 2 spawns observed,
   each completing 1 story (matched by the `--max-continues 1` inner cap).
2. **`mode.ts` owns iteration advance** — the counter incremented from
   1 → 2 between spawns without ever firing a Stop hook handler. The
   upstream Copilot pwsh dispatch bug doesn't block iteration advance
   anymore because no hook is in the critical path.
3. **`outerLoopOwned` flag works correctly** — set to `true` during the
   run (would suppress Stop hook if it fired), cleared to `false` on
   loop exit so a subsequent `omcp ralph --resume` doesn't think the
   outer loop is still active.
4. **Max-exhaustion preserves state for resume** — `active: true` +
   `iteration: 2` means a follow-up `omcp ralph --resume` would pick up
   at iteration 3 to finish story 3 (this resume path is verified in
   deterministic tests; live verification deferred to v1.7).
