## What NOT to do (lessons from previous sessions)

1. **Don't tag before live smoke evidence.** v1.3-v1.5 shipped tags while the user-facing
   iteration-advance bug persisted because deterministic tests didn't catch it. v1.6
   established the gate; never lower it again.

2. **Don't push carry-forward without an ADR**. v1.5 silently deferred DEP0190; took
   to v1.7 to fix. If you can't solve in current version, write an ADR that explains
   why specifically and don't just say "v1.8".

3. **Don't try single-agent on hard problems**. DEP0190 single-agent attempt in v1.5
   failed; multi-direction team (3 parallel agents) in v1.7 converged on same solution.
   The "3+ parallel directions" principle works.

4. **Don't trust hook reminders that say "Edit operation failed"** when the tool result
   says "updated successfully". The hook reminders have been noisy across many sessions;
   the tool result is authoritative. If a real edit fails, the next operation (read /
   tsc / test) will surface it.

5. **Don't run `cd` to a tmp dir then run a follow-up command without `cd` back to
   the repo root**. Background bash commands lose cwd. Use absolute paths or chain
   commands with `&&`.

6. **Don't go to file an upstream issue via the runjiashi_microsoft account** — it's
   an EMU account, blocked from public github/copilot-cli. User handles upstream
   filing personally.
