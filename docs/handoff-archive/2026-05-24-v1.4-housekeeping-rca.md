## Bug details (Lane 1+2 — VERIFIED via code-path read, NOT smoke)

**File**: `src/cli/commands/mode.ts`

**Lines 257-259** (pre-spawn snapshot):
```ts
const ralphSnapshot  = opts.mode === "ralph" ? readRalphState() : null;
const prdStatusSnapshot = opts.mode === "ralph" ? getPrdCompletionStatus() : null;
```
These are read BEFORE `spawnSyncCrossPlatform` at line 261. The PRD has 0 stories complete at this moment (run hasn't happened). So `prdStatusSnapshot.allComplete` is always `false` for a first-time PRD.

**Lines 293-307** (post-exit conditional restore):
```ts
const shouldClear = (prdStatusSnapshot?.allComplete ?? false) || ralphSnapshot.architectApproved;
if (!shouldClear && ralphSnapshot) {
  writeRalphState(ralphSnapshot);  // ← writes back iteration:1, spawn-time lastFiredAt
}
```
Because `prdStatusSnapshot` was captured pre-spawn, it can never reflect run-time PRD completion. `shouldClear` is always `false` for clean runs → state always gets restored.

**Fix direction**:
```ts
// AFTER spawn returns:
const postRunPrd = getPrdCompletionStatus();
const postRunRalph = readRalphState();
const shouldClear = postRunPrd.allComplete || postRunRalph?.architectApproved;
// ... use post-run state for the decision
```

**Test gap**: `ralph-state-crash-recovery.integration.test.ts` test 1 passes because it pre-writes complete PRD BEFORE calling `runMode`. So `prdStatusSnapshot.allComplete` is `true` at spawn time → `shouldClear=true` → state cleared. This is not the live scenario.

**Real test needed**: PRD pre-incomplete → mock spawn that completes the PRD → assert state cleared. This test will FAIL on current main; that's the proof.
