## 1. Goal

Bring the omc/omx multi-agent orchestration experience to **GitHub Copilot CLI
1.0.32+**, with acceptance criterion *"UX consistent with omc and omx"*:

- Same vocabulary (`autopilot`, `ralph`, `team`, `ultraqa`, `ccg`, `plan`, `hud`, `cancel`, …)
- Same install ergonomics (`npm install -g oh-my-copilot` then `omcp setup`)
- Same in-CLI feel (slash commands under a single namespace, agent catalog, MCP-backed state)
- Native to Copilot's surface (uses `/fleet`, `/delegate`, `/tasks`, MCP, plugin marketplace)

## 2. Non-goals (v0.1)

- Web dashboard / GUI
- Compatibility shims for Claude-only tools (`Task`, `TaskCreate`, `ToolSearch`, etc.) — skills must not reference them
- Hosting Codex-only sparkshell internals (stays in omx)
- Telemetry / analytics service
- Auto-upgrade flows beyond `omcp setup --update`

## 3. Architecture

### 3.1 Top-level layout (single-package monorepo, omx-style)

```
oh-my-copilot/
├─ src/                          TypeScript source
│  ├─ cli/                       commander entry + subcommands
│  ├─ runtime/                   model routing, env, paths
│  ├─ mcp/                       MCP servers (state, trace, notepad)
│  ├─ hooks/                     pre/post tool hooks
│  ├─ state/                     SQLite-backed session state
│  ├─ scripts/                   build helpers, plugin-mirror sync, doctor
│  └─ __tests__/                 vitest specs
├─ crates/
│  └─ omcp-explore-harness/      Rust hot-path harness (optional at install time)
├─ agents/                       19 agent .md files
├─ skills/                       ~30 skill .md files
├─ prompts/                      reusable prompt fragments
├─ templates/                    scaffolding templates
├─ plugins/oh-my-copilot/        BUILD MIRROR (generated by sync-plugin-mirror.ts)
├─ .claude-plugin/plugin.json    Copilot-compatible plugin manifest
├─ .agents/plugins/marketplace.json   plugin marketplace listing
├─ .mcp.json                     MCP server registry
├─ AGENTS.md                     Copilot CLI instructions for this repo
├─ CLAUDE.md                     Claude Code instructions for this repo
├─ README.md
├─ LICENSE                       MIT
├─ Cargo.toml                    Rust workspace root
├─ package.json                  npm package (single)
└─ tsconfig.json
```

### 3.2 Components

#### A. `omcp` CLI (TypeScript, `src/cli/`)

Subcommands (M1 set in **bold**, rest M3-M5):

- **`setup`** — install plugin into `~/.copilot/installed-plugins/oh-my-copilot/oh-my-copilot/`, write/refresh `~/.copilot/config.json` plugin entries, optionally register MCP servers
- **`doctor`** — check Copilot CLI version, plugin install state, MCP reachability, model availability, permissions
- **`ask`** — `omcp ask <claude|gpt|auto> "<prompt>"` — spawns `copilot -p` with chosen `--model` and `--allow-all-tools`
- **`team`** — `omcp team N:<agent> "<task>"` — spawns N tmux panes each running `copilot --agent <agent>`
- `version` — print version
- `mcp-setup` — interactive MCP server picker
- `release` — internal: publish + tag
- `learn` — emit `learner` skill output

CLI dispatched via `commander`, mirroring omc's `bridge/cli.cjs` ergonomics.

#### B. Plugin bundle (markdown + JSON, `agents/` + `skills/` + `.claude-plugin/`)

Copilot CLI loads plugins from `~/.copilot/installed-plugins/<marketplace>/<plugin>/`. The plugin manifest (`.claude-plugin/plugin.json`) points to:

- `agents/` — one `<name>.md` per agent with frontmatter (`description`, `model`, `tools`)
- `skills/` — one `<name>/SKILL.md` per skill with frontmatter (`name`, `description`, `triggers`)
- `.mcp.json` — MCP server registry

Slash invocation: `/oh-my-copilot:<skill-name>` (Copilot derives the namespace from the plugin name).

#### C. MCP servers (`src/mcp/`)

- `state-server` — `state_read`, `state_write`, `state_clear`, `state_list_active`, `state_get_status`
- `notepad-server` — `notepad_read`, `notepad_write_priority`, `notepad_write_working`, `notepad_write_manual`, `notepad_prune`, `notepad_stats`
- `trace-server` — `trace_summary`, `trace_timeline`
- `memory-server` — `project_memory_read`, `project_memory_write`, `project_memory_add_note`, `project_memory_add_directive`

All servers ship as compiled `dist/mcp/<name>.js` and are referenced by `.mcp.json` via `${PLUGIN_ROOT}/dist/mcp/<name>.js`.

#### D. Rust harness (`crates/omcp-explore-harness/`)

Optional Rust binary `omcp-explore` providing fast directory walk + grep/glob dispatch (mirrors omx's `omx-explore-harness`). The TypeScript runtime detects its presence and falls back to a pure-TS implementation if absent — ensures `omcp` works with or without a Rust toolchain installed.

#### E. Model routing (`src/runtime/model-routing.ts`)

Every agent declares a `model:` map:

```yaml
model:
  claude: claude-opus-4.7
  gpt:    gpt-5.4
```

Routing decision at agent-spawn time:

1. If `OMCP_MODEL_FAMILY=claude|gpt`, use that family's value
2. Else read `~/.copilot/config.json`'s `model` field — infer family from prefix
3. Else default to `claude` (matches Copilot's default)

Caller may override at invocation: `omcp ask gpt "..."` or `--model gpt-5.4`.

### 3.3 Data flow

#### Install flow (`omcp setup`)

```
omcp setup
  ├─ verify copilot CLI on PATH (>= 1.0.32)
  ├─ ensure ~/.copilot/ exists
  ├─ create ~/.copilot/marketplaces/oh-my-copilot.json from .agents/plugins/marketplace.json
  ├─ copy/symlink package root into ~/.copilot/installed-plugins/oh-my-copilot/oh-my-copilot/
  ├─ merge ~/.copilot/config.json plugin entry
  ├─ merge ~/.copilot/mcp-config.json from .mcp.json (with ${PLUGIN_ROOT} substitution)
  └─ print success summary + next steps
```

#### Skill invocation flow

```
user types /oh-my-copilot:ralph "..." in Copilot
  -> Copilot loads skills/ralph/SKILL.md
  -> Skill prompt runs against current model (Claude or GPT, per /model)
  -> Skill may invoke /fleet for parallel subagents, or MCP tools (state, trace, …)
  -> Skill emits Markdown back to the Copilot transcript
```

#### `omcp team` flow

```
omcp team 4:executor "Implement X"
  ├─ create tmux session 'omcp-team-<uuid>'
  ├─ spawn 4 panes: copilot --agent executor -i "Implement X (worker N)"
  ├─ each pane logs to .omcp/state/sessions/<sid>/worker-N.log
  └─ aggregator pane tails all workers and writes consensus to notepad
```

### 3.4 State & persistence

Per-repo state directory: `.omcp/` (mirrors omc's `.omc/`).

```
.omcp/
├─ state/                       SQLite (better-sqlite3) for session state
│  └─ sessions/<sessionId>/     per-session scratch
├─ notepad.md                   shared notepad
├─ project-memory.json          structured project memory
├─ plans/                       active plans
├─ research/                    research artifacts
└─ logs/                        runtime logs
```

User-global state: `~/.copilot/` (Copilot owns this; omcp only writes its own plugin entry and MCP config keys).

## 4. Error handling

| Class                              | Behavior                                                      |
| ---------------------------------- | ------------------------------------------------------------- |
| `copilot` not on PATH              | `omcp doctor` fails with install link; setup aborts with clear error |
| Copilot version < 1.0.32           | Warn, allow `--force` to proceed                              |
| Plugin already installed (newer)   | `omcp setup` prompts before downgrade (or `--force`)          |
| MCP server fails to spawn          | Log + degrade gracefully (skills that don't need that server still work) |
| Rust harness missing               | TS fallback path used; no user-visible error                  |
| `OMCP_MODEL_FAMILY` invalid        | Hard fail with list of valid values                           |
| `tmux` missing on Windows for `team` | Fall back to background processes with file-tail aggregator   |

Every CLI exit code is documented in `docs/architecture/exit-codes.md` (M1).

## 5. Testing strategy

- **Unit (vitest):** `src/runtime/`, `src/cli/`, `src/mcp/` modules — pure logic
- **Integration:** `src/__tests__/setup-flow.test.ts` against a fixture `~/.copilot/` directory (TMP override via `OMCP_HOME`)
- **Plugin-bundle invariants:** `verify-plugin-bundle.ts` checks `plugins/oh-my-copilot/` matches source-of-truth (agents/ + skills/ + .claude-plugin/) byte-for-byte
- **Catalog SSOT:** `verify-catalog.ts` parses every agent/skill frontmatter and asserts dual-model declarations + no Claude-only tool references
- **Rust:** `cargo test -p omcp-explore-harness`
- **E2E (manual, M5):** `omcp setup` on a clean VM, then a smoke-test session in Copilot validating one skill from each category

CI matrix: `{ubuntu-latest, macos-latest, windows-latest} × node 20/22`.

## 7. Decisions log

| # | Decision                                                | Rationale                                                                          |
| - | ------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| 1 | Single-package monorepo (omx-style)                     | User chose "镜像 omx"; aligns publishing lifecycle                                |
| 2 | TS + Rust hybrid                                        | User choice; matches omx pattern                                                   |
| 3 | Manual rewrite of every skill/agent                     | Highest quality; avoids Claude-only tool leakage                                  |
| 4 | Dual model paths (Claude + GPT mirrors) selected at runtime | Copilot supports both vendors; agent files describe both                          |
| 5 | Slash prefix `/oh-my-copilot:<name>`                    | Matches Copilot's plugin-name-derived namespace convention                         |
| 6 | Repo dir `oh-my-copilot-r2` to avoid colliding with parallel session | Disk-only suffix; published name remains `oh-my-copilot`                          |
| 7 | M0→M5 milestone delivery (not big-bang)                 | User choice; manageable review cadence                                             |
| 8 | Plugin mirror (`plugins/oh-my-copilot/`) is generated, not edited | Single source of truth at repo root; mirror exists for plugin-marketplace install |

## 8. Risks

| Risk                                                           | Mitigation                                                                      |
| -------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| Copilot plugin manifest format diverges from `.claude-plugin/` | Doctor probes the actual `~/.copilot/installed-plugins/omc/oh-my-claudecode/` shape and we mirror it; CI test loads our manifest with `copilot plugin list` against a clean home |
| Rust toolchain unavailable on user machines                    | Pure-TS fallback for every Rust-backed feature; `omcp setup` reports degradation but does not fail |
| Skill catalog grows beyond review capacity                     | Catalog SSOT linter + dual-review (writer pass + verifier pass); enforced via `verify-catalog`     |
| Parallel session diverges from r2 design                       | r2 owns this spec; merge-or-discard decision left to user post-M2                |
