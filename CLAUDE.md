# oh-my-copilot — Working Notes for Claude Code

This repository implements **omcp**, the GitHub Copilot CLI sibling of
oh-my-claudecode and oh-my-codex.

## Authoritative reference projects

- omc (oh-my-claudecode) at `C:\Users\runjiashi\.claude\plugins\cache\omc\oh-my-claudecode\4.9.3\` — read-only reference
- omx (oh-my-codex) at `C:\Users\runjiashi\_refs\oh-my-codex\` — read-only reference

When porting a skill/agent, open the omc/omx counterpart side-by-side and
**rewrite manually** (do not blind-copy) — strip Claude-only tool names,
re-target Copilot's tool surface, and emit dual-model recommendations.

## Tooling differences to respect

| Claude Code tool         | Copilot CLI equivalent                                |
| ------------------------ | ----------------------------------------------------- |
| `Task` agent dispatch    | `/fleet`, `/delegate`, `/tasks`                       |
| `TaskCreate` / TodoWrite | `/tasks` UI + plain checklists in skill outputs       |
| MCP servers (Anthropic)  | Same MCP protocol; configure via `~/.copilot/mcp-config.json` |
| `EnterPlanMode`          | `--mode plan` / `/plan` slash                         |
| Skill tool               | `/skills` slash + `/oh-my-copilot:<name>` invocation  |
| Bash                     | Bash tool exists in Copilot (subject to `--allow-tool`) |
| `WebSearch` (built-in)   | Not available — use shell `curl`/`gh` or document-specialist MCP |

## Model routing

Every agent declares a `model_family` pair:

```yaml
model:
  claude: claude-haiku-4.5 | claude-sonnet-4.6 | claude-opus-4.7
  gpt:    gpt-5-mini       | gpt-5.2           | gpt-5.4
```

omcp's runtime reads `OMCP_MODEL_FAMILY` env var (`claude` | `gpt` | `auto`)
or falls back to `~/.copilot/config.json` `model` field to pick which side.

## Repo conventions

- Source of truth lives at `src/`, `agents/`, `skills/`, `prompts/`, `templates/`
- `plugins/oh-my-copilot/` is a build mirror — never edit by hand; regenerated by `scripts/sync-plugin-mirror.ts`
